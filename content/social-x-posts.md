## PLATFORM-SPECIFIC TIPS (Posts 16–20)

---

**Post 16 — Platform Tip**
If your agent keeps asking the same clarifying questions, you don't have a prompt problem — you have a memory problem. Fix: create a preferences file your agent reads at the start of every session. 15 minutes of setup. Hours saved per week. Start there.

---

**Post 17 — Platform Tip**
The simplest memory system that actually works: 3 files. One for user preferences. One for past decisions and outcomes. One for ongoing project context. Your agent reads all 3 on startup. That's it. No vector database required to get 80% of the value.

---

**Post 18 — Platform Tip**
If you're building with Claude, GPT-4, or Gemini and wondering why your agent keeps "forgetting" — it's not the model, it's your architecture. The model can only work with what you inject. Memory is a system design problem, not an AI problem. agentawake.com

---

**Post 19 — Platform Tip**
Stop stuffing your system prompt with everything you want your agent to know. Instead: build a retrieval step. Let the agent pull relevant context at runtime based on what's actually needed. Leaner prompts, smarter behavior, faster responses. Architecture > prompting.

---

**Post 20 — Platform Tip**
The one question that will improve every AI agent you build: "What should this agent know tomorrow that it learned today?" Answer that, write the code that captures it, and you'll be ahead of 95% of people building automation right now. Start with one memory type.
